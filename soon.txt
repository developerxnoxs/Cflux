STRUKTUR PROYEK
---------------
src/
├── lexer/       – Tokenizer (indentation-aware, f-string, operator |>)
├── parser/      – Recursive-descent parser → AST
├── compiler/    – AST → bytecode (single-pass)
├── vm/          – Stack-based VM + coroutine scheduler (libuv)
├── gc/          – Mark-and-sweep garbage collector
├── object/      – Runtime types: String, List, Dict, Closure, Class, Future
├── runtime/     – Built-in functions
├── api/         – Public C API (flux_vm_new, flux_eval, dll.)
└── main.c       – CLI entry point

stdlib/          – Module .so: io, math, time, fs, os, sys, json, socket, async
extension/       – Module .so: http (libcurl), mysql, postgresql, ws (WebSocket)
include/flux/    – Public headers
tests/           – .flx test scripts + C-level unit tests

[ORANYE] IMPLEMENTASI SETENGAH JALAN
--------------------------------------

8. match statement / pattern matching
   Lokasi : src/compiler/compiler.c:2116
   Status : Hanya melakukan equality check (subject == pattern).
            TIDAK ADA:
            - Destructuring list/dict dalam pattern
            - Binding variable ke pattern (case Point(x, y):)
            - Wildcard _ sebagai catch-all sejati
            - Guard clause (case x if x > 0:)
            - Match berdasarkan tipe
            - Pattern matching untuk enum (karena enum sendiri belum proper)

9. struct
   Lokasi : src/compiler/compiler.c:2171
   Status : Diimplementasikan dengan mem-reuse OP_CLASS — struct adalah
            class biasa dengan auto-generated init dari deklarasi field.
            Tidak ada perbedaan semantik nyata antara struct dan class:
            - Tidak ada value semantics (copy-by-value)
            - Tidak ada memory layout yang berbeda
            - Tidak ada batasan inheritance atau method

10. enum
    Lokasi : src/compiler/compiler.c:2296
    Status : Dicompile menjadi OP_CREATE_DICT — enum hanyalah dict biasa
             {"Red": 0, "Green": 1, "Blue": 2}.
             Tidak ada tipe Enum di VM level, tidak bisa dipakai di match
             untuk pattern matching enum yang proper.

11. async/await — awaitable non-standar
    Lokasi : src/vm/vm.c:2035
    Status : OP_AWAIT hanya menangani objek tipe Future dan Coroutine.
             Await pada tipe lain langsung error runtime.
             Tidak ada protokol awaitable generik (seperti __await__ di Python)
             sehingga tidak bisa membuat custom awaitable object.

12. Circular import / Module isolation
    Lokasi : src/vm/vm.c (fungsi do_import)
    Status : import dan from ... import bekerja untuk kasus normal, tapi
             tidak ada deteksi atau proteksi circular dependency.
             Program dengan import melingkar akan stack overflow atau
             infinite loop tanpa pesan error yang jelas.


[KUNING] BUG / PERILAKU TIDAK KONSISTEN
-----------------------------------------

13. json.decode — Unicode rusak
    Lokasi : stdlib/json/json_module.c:193
    Status : Karakter non-ASCII dalam JSON string (emoji, karakter CJK,
             Arab, Latin berdiakritik) dibuang diam-diam dan diganti '?'.
             Komentar kode: /* skip non-ASCII */
             Program yang memroses JSON dari API internasional akan mendapat
             data rusak TANPA error atau peringatan apapun.

14. os.listdir — tidak ada di Windows
    Lokasi : stdlib/os/os_module.c:56
    Status : Dikompilasi hanya di dalam #ifndef _WIN32 guard.
             Di Windows fungsi ini tidak terdaftar sama sekali —
             tidak return error, tidak ada fallback.

15. Error handling tidak konsisten antar modul
    Lokasi : Berbagai file di stdlib/
    Status : socket dan http return dict {"ok": false, "error": "pesan"} → bisa di-handle.
             fs, json, time, dan modul lain hanya return null tanpa info apapun.
             Tidak ada standar error reporting di stdlib, sehingga kode
             error handling tidak bisa ditulis secara seragam.

16. object_print untuk Future
    Lokasi : src/object/object.c:191
    Status : Hanya print literal "<future>" — tidak ada informasi state
             (pending / resolved / rejected) atau nilai hasil setelah resolved.
